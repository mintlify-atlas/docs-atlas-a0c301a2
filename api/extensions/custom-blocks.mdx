---
title: "CustomBlocks — CPE extension API"
description: "API reference for the CustomBlocks extension: negotiate custom block support levels with a client and register the support-level packet handler."
---

`CustomBlocks` manages the handshake that establishes how many custom block IDs a client supports. When this extension is negotiated, the constructor automatically registers a `CustomBlockSupportLevel` packet handler so the server can read the client's reported support level. You then call `sendSupportLevel` to tell the client which level the server is operating at.

This class extends `BaseExtension`, so every instance exposes a `client` property pointing to the owning `Client` and a `version` property for the negotiated extension version.

**CPE name:** `CustomBlocks`

```javascript
import { extensions } from "classicborne-server-protocol"
```

```javascript
const ext = client.extensions.get("CustomBlocks")
```

## Methods

### `sendSupportLevel`

```javascript
ext.sendSupportLevel(supportLevel)
```

Sends a `CustomBlockSupportLevel` packet (packet ID `0x13`) declaring the server's supported custom block level.

<ParamField path="supportLevel" type="number" required>
  The custom block support level to declare. Level `1` is the only level defined by the current CPE spec. Send the same level the client reported to complete the handshake.
</ParamField>

<Note>
  The constructor registers a packet handler that reads the client's own `CustomBlockSupportLevel` packet. Call `sendSupportLevel` after receiving the client's level — typically in the `clientConnected` handler — to complete the negotiation.
</Note>

```javascript
server.on("clientConnected", (client) => {
  if (client.extensions.has("CustomBlocks")) {
    const ext = client.extensions.get("CustomBlocks")
    ext.sendSupportLevel(1)
  }
})
```
