`BlockDefinitionsExtended` builds on the base block definition packet by giving you independent texture indices for each of the six faces and full control over the block's bounding box. Use it instead of `BlockDefinitions` when you need non-cubic hitboxes or different artwork on the left, right, front, and back faces.

This class extends `BaseExtension`, so every instance exposes a `client` property pointing to the owning `Client` and a `version` property for the negotiated extension version.

**CPE name:** `BlockDefinitionsExt`

```javascript
import { extensions } from "classicborne-server-protocol"
```

```javascript
const ext = client.extensions.get("BlockDefinitionsExtended")
```

## Static properties

**`BlockDefinitionsExtended.defineBlockPacketIdentifier`** ŌĆö `0x25`. The packet type byte written at the start of every `defineBlock` packet.

## Methods

### `defineBlock`

```javascript
ext.defineBlock(block)
```

Sends an extended block definition packet. The extended format replaces the single `sideTexture` field from `BlockDefinitions` with four directional texture fields, and adds six bounding-box fields. All fields are optional and default to sensible values.

<ParamField path="block.collision" type="number" default="2">

<ParamField path="block.speed" type="number" default="128">

<ParamField path="block.bottomTexture" type="number" default="0">

<ParamField path="block.transmitLight" type="number" default="0">

```javascript
server.on("clientConnected", (client) => {
  if (client.extensions.has("BlockDefinitionsExtended")) {
    const ext = client.extensions.get("BlockDefinitionsExtended")
    ext.defineBlock({
      id: 60,
      name: "Half Slab",
      collision: 1,
      topTexture: 5,
      leftTexture: 6,
      rightTexture: 6,
      frontTexture: 6,
      backTexture: 6,
      bottomTexture: 5,
      minX: 0,
      minY: 0,
      minZ: 0,
      maxX: 16,
      maxY: 8,
      maxZ: 16,
      draw: 0,
    })
  }
})
```
