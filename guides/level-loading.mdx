title: "Loading and sending level data to clients"
description: "How to format level buffers, compress them with processLevel, and stream the data to clients using loadLevel with pre-processing for efficiency."
---

Before a client can play, you must send them the full level. The library handles the low-level protocol details ‚Äî packet framing, chunking, and progress reporting ‚Äî but you are responsible for building the level `Buffer` and deciding when and how to compress it.

## Level data format

Level data is a flat `Buffer` where each byte is a block type ID. Blocks are stored in YZX order: the index for a block at `(x, y, z)` in a world with dimensions `(sizeX, sizeY, sizeZ)` is:

```
index = x + sizeZ * (z + sizeX * y)
```

You can write a helper to make this easier:

```js server.mjs
const bounds = { x: 256, y: 256, z: 256 }
const level = Buffer.alloc(bounds.x * bounds.y * bounds.z)

function setBlock(x, y, z, type) {
  level[x + bounds.z * (z + bounds.x * y)] = type
}
```

Common block type IDs: `0` Air, `1` Stone, `2` Grass, `3` Dirt, `4` Cobblestone, `7` Bedrock.

## Compressing level data with `processLevel`

Before sending the level to a client, you must compress it. The `processLevel` function prepends a 32-bit block count, then gzip-compresses the result, and returns a `Promise<Buffer>`.

```js
processLevel(data, x, y, z)
```

| Parameter | Type | Description |
| --- | --- | --- |
| `data` | `Buffer` | The raw level buffer containing block type IDs. |
| `x` | `number` | Width of the level (X dimension). |
| `y` | `number` | Height of the level (Y dimension). |
| `z` | `number` | Depth of the level (Z dimension). |

Access it through the server instance:

```js server.mjs
const processed = server.utils.processLevel(level, bounds.x, bounds.y, bounds.z)
```

## Sending the level with `loadLevel`

Call `client.loadLevel` to stream the compressed level data to a client. The method sends the level initialize packet, streams the data in 1024-byte chunks, then sends the finalize packet with the world dimensions.

```js
client.loadLevel(data, x, y, z, processed?, callback?, preFinalize?)
```

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `data` | `Buffer` | ‚Äî | The level data. Pass the raw buffer or a pre-processed one. |
| `x` | `number` | ‚Äî | Width of the level. |
| `y` | `number` | ‚Äî | Height of the level. |
| `z` | `number` | ‚Äî | Depth of the level. |
| `processed` | `boolean` | `false` | When `true`, skips internal compression. Use this with a buffer already returned by `processLevel`. |
| `callback` | `function` | ‚Äî | Called after the finalize packet is sent. |
| `preFinalize` | `function` | ‚Äî | Called just before the finalize packet is sent. |

## Pre-processing for multiple clients

Compressing a level takes time and CPU. If many clients connect, you do not want to compress the same data repeatedly. Call `processLevel` once at startup and reuse the result for every client.

The following example from `minimum.mjs` demonstrates this pattern:

```js minimum.mjs
import { Server } from "classicborne-server-protocol"
const server = new Server(25565)

const bounds = { x: 512, y: 512, z: 512 }
const emptyLevelBuffer = server.utils.processLevel(
  Buffer.alloc(bounds.x * bounds.y * bounds.z),
  bounds.x,
  bounds.y,
  bounds.z
)

server.on("clientConnected", async (client, authInfo) => {
  client.serverIdentification("Empty", "So cool!", 0x00)
  client.loadLevel(await emptyLevelBuffer, bounds.x, bounds.y, bounds.z, true)
  client.configureSpawn(0, authInfo.username, 0, 0, 0, 0, 0)
})
```

<Note>
`server.utils.processLevel` returns a `Promise`. Declare the call outside the connection handler so the compression runs once, then `await` it inside the handler when you need the result.
</Note>

## Using `preFinalize` and `callback`

The `preFinalize` hook runs after all data chunks are sent but before the finalize packet is delivered. Use it to configure the environment or spawn entities while the client is still in the level-loading screen.

The `callback` hook runs after the finalize packet is sent, signalling that the client has fully entered the world and is ready to receive gameplay packets.

```js server.mjs
client.loadLevel(level, bounds.x, bounds.y, bounds.z, false,
  () => {
    // callback: client is fully in the world
    client.configureSpawn(0, authInfo.username, 128, 32, 128, 0, 0)
  },
  () => {
    // preFinalize: set environment before the client finalizes the level
    // e.g. configure sky color, fog, etc. via extensions
  }
)
```

<Tip>
Use `preFinalize` to send environment configuration packets before the client finalizes the level. This ensures the environment is already set when the client first sees the world.
</Tip>
